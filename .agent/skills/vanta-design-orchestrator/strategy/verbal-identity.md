# Verbal Identity — Voz, Tono y Contenido

> El logo es la cara. La voz es la personalidad. El tono es el estado de ánimo.
> Una marca sin voz consistente es una cara sin expresión.

---

## 1. Voz de Marca (Constante)

La voz es la personalidad lingüística de la marca. No cambia con el contexto.

### Dimensiones de la Voz

| Dimensión | Espectro | Ejemplo |
|:----------|:---------|:--------|
| **Formal ↔ Informal** | ¿Usamos "usted" o "tú"? | Fintech → formal-medium. D2C → casual |
| **Serio ↔ Divertido** | ¿Hacemos chistes? | Healthcare → serio. Gaming → divertido |
| **Respetuoso ↔ Irreverente** | ¿Cuestionamos normas? | B2B → respetuoso. Rebelde → irreverente |
| **Concreto ↔ Abstracto** | ¿Datos exactos o metáforas? | SaaS → concreto. Luxury → abstracto |
| **Entusiasta ↔ Sobrio** | ¿Muchos signos de exclamación? | D2C → entusiasta. Enterprise → sobrio |

### Definición de Voz
```
[Brand] suena como [referencia cultural 1] con toques de [referencia cultural 2],
pero más [adjetivo] y menos [adjetivo opuesto].
```
**Ejemplo:** "Mailchimp suena como un amigo que sabe de marketing, pero sin ser intenso. Es experto pero accesible."

## 2. Tono de Marca (Variable)

El tono se adapta al contexto y canal. La voz es siempre la misma; el tono cambia.

| Contexto | Tono recomendado | Ejemplo de voz (misma marca) |
|:---------|:-----------------|:-----------------------------|
| **Homepage** | Inspiracional, aspiracional | "Imagina lo que puedes lograr" |
| **Error 404** | Humorístico, empático | "Ups, esto no debería pasar" |
| **Email de bienvenida** | Cálido, entusiasta | "¡Nos alegra tenerte aquí!" |
| **Soporte técnico** | Paciente, claro, directo | "Vamos a resolver esto paso a paso" |
| **Términos de servicio** | Formal, preciso | No aplicar tono — solo claridad |
| **Redes sociales** | Coloquial, conversacional | "¿Qué opinan de esta feature?" |
| **Comunicado de crisis** | Serio, transparente, humano | "Nos equivocamos y esto es lo que haremos" |

## 3. Jerarquía de Contenido

### Sistema de Headings
| Nivel | Propósito | Tipografía | Longitud máx |
|:------|:----------|:-----------|:-------------|
| **H1** | Título principal, valor único | Display/Bold, 40-64px | 8 palabras |
| **H2** | Secciones principales | SemiBold, 28-40px | 10 palabras |
| **H3** | Sub-secciones | SemiBold, 20-28px | 12 palabras |
| **H4+** | Detalles dentro de sub-secciones | Bold, 16-20px | 15 palabras |
| **Body** | Contenido principal | Regular, 16-18px | 45-75 caracteres/línea |
| **Small/Caption** | Notas, metadata | Regular, 12-14px | N/A |
| **Microcopy** | Labels, CTAs, errores | Bold, 14-16px | 3-5 palabras |

### Reglas editoriales
1. **Una idea por párrafo.** Si un párrafo tiene dos ideas, dividir.
2. **Párrafos de 3-4 líneas máximo.** Más de 5 líneas se salta.
3. **Listas con viñetas para 3+ items.** El cerebro procesa listas 50% más rápido.
4. **Voz activa.** "El sistema procesa" no "El procesamiento es realizado por el sistema".
5. **Evitar jargon.** Si usas un término técnico, defínelo en la primera mención.

## 4. Escritura de CTAs (Call to Action)

| Regla | Correcto | Incorrecto |
|:------|:---------|:-----------|
| **Verbo + beneficio específico** | "Empieza tu prueba gratis" | "Get Started" |
| **Sin jargon** | "Descarga el reporte" | "Accede al asset descargable" |
| **Urgente pero no agresivo** | "Plazas limitadas — reserva hoy" | "¡COMPRA AHORA!" |
| **Primera persona (opcional)** | "Crea mi cuenta" | "Registrarse" |
| **Específico sobre el outcome** | "Aumenta tus ventas en 30 días" | "Mejora tu negocio" |

---

## 5. Visualización de Datos — Pautas de Contenido

| Regla | Por qué | Cómo |
|:------|:--------|:-----|
| **Una pregunta por gráfico** | Claridad cognitiva | El título debe ser la pregunta que responde |
| **Data-ink ratio máximo** | No distraer | Eliminar gridlines, decoración, 3D falso |
| **Datos reales > placeholder** | Credibilidad | Si no hay datos reales, no mostrar stats |
| **Colorblind-safe palette** | Inclusión | Usar patrones además de color |
| **Anotaciones contextuales** | Sin asumir conocimiento | Explicar qué significa cada dato |

## 6. Glosario de Marca

Toda marca debe tener un glosario de:
- **Términos propietarios:** Palabras inventadas o con significado específico para la marca
- **Términos prohibidos:** Palabras que la marca nunca usa
- **Términos preferidos:** Alternativas a términos comunes que la marca usa

**Ejemplo:**
```
Propietario: "VantaVerse" (nuestro ecosistema)
Prohibido: "users" → usar "creators"
Preferido: "plan" → "growth tier"
```
